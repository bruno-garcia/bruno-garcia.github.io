    Pitfalls of Unoptimized NuGet Packages

     A bunch of code here

    You add a NuGet package to your project. Was it compiled with optimizations? There’s a chance it wasn’t. That’s because dotnet pack and dotnet publish by default use Configuration=Debug. This can affect the size and performance of the compiled assemblies.

    https://twitter.com/khellang/status/1229171900141756416

    <ItemGroup>
  <PackageReference Include="UnoptimizedAssemblyDetector" Version="0.0.5">
    <PrivateAssets>all</PrivateAssets>
  </PackageReference>
</ItemGroup>
